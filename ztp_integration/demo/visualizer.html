🤖 How GemmaBot Controls the Robot

Semantic → Geometric Bridge + ZTP Physics Guardrail

🧠 The Problem

Gemma 4 is a vision-language model. It sees images, not coordinates. It can say "pick cracked_cup" but it can't say "move x=330 y=180" — and we shouldn't want it to!

Solution: Two-layer architecture:

Gemma 4:  "pick cracked_cup"    ← semantic (names what it sees)
  ↓
Sim:      cracked_cup → (330,180) ← geometric (resolves name to position)
  ↓
Physics:  20px step toward target  ← simple motion (move along vector)
  ↓
ZTP:      force=0.8N < 1.2N ✓      ← safety check (block if unsafe)

📍 Movement Path: Gripper → Cracked Cup

Each red dot is one tick. Gripper moves 20px/tick along the straight-line vector. Gemma 4 re-decides EVERY tick from the current image — so if the cup moves, it tracks it.
